# Tenant contact forms and visual settings

Tenant public contact forms are styled from the shared tenant visual settings so the contact page matches the public portfolio and tenant admin preview. The contact page uses card-style form panels, first-party CAPTCHA, async submission status, and the tenant text/content surface variables.

Tenant Admin > Settings is rendered in the tenant admin shell, not the platform admin shell. Page, heading, text, menu, top-bar, content, and artwork-card settings are applied as CSS variables in both public tenant pages and tenant admin pages so the admin screen previews the tenant site instead of drifting into platform styling.
